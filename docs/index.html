How it works

Three steps. That's it.

Step 01
◎
Drop your binary
Drag in any executable — .exe, ELF, .dylib, firmware, WASM. Softy detects the format, architecture, and compiler automatically, then starts analysis immediately.
→
Step 02
✦
AI reads it for you
Ghidra decompiles. AI renames every variable and function to something meaningful, recovers types, identifies algorithms, and adds explanatory comments. Functions stream in as they're processed.
→
Step 03
⬡
Edit and rebuild
Edit the decompiled C code in Monaco — real-time syntax checking, full autocomplete. Hit compile. Clang rebuilds. Your modified binary is ready. Or export to VS Code for a full IDE workflow.

What does FUN_004021a8 do? It keeps getting called before network operations.
✦ Softy AI
This is a TLS/SSL context initialization function. It:

1. Calls SSL_CTX_new() with TLS_method()
2. Sets certificate verification to SSL_VERIFY_PEER
3. Loads the system CA bundle via SSL_CTX_set_default_verify_paths()

It's called before every new connection to ensure a fresh context. I've renamed it to initSslContext() and recovered the SSL_CTX* return type.
Are there any certificate validation bypasses?
✦ Softy AI
⚠️ Found one. In verifyCallback() at 0x00402a14, the verification result is overridden unconditionally — it always returns 1 (accept) regardless of the actual cert check outcome.

This is a classic certificate pinning bypass. The binary will accept any certificate, including self-signed or expired ones.
